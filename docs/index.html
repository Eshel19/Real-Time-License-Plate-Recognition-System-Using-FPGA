Control Plane

AHIM: Accelerator Host Interface Manager

The AHIM transforms the FPGA from a passive peripheral into a self-managed co-processor. It handles the quirks of the Avalon bridge (splitting 128-bit writes into four 32-bit chunks) and manages the OCR engine autonomously.

RX Unit

Configurable

Compute Plane

Sliding Window CNN Accelerator

A pipelined implementation of a convolutional neural network designed specifically for streaming text recognition. Instead of batch processing, it scans the image strip column-by-column.

  • Parallel Compute Engine

    16 MultiMultiplier Engines operating in parallel. Dynamically switches between Convolution and Fully Connected modes to maximize DSP usage.

  • Data Management Unit (DMU)

    Manages sliding window buffers and feature maps. No external memory access required during inference.

Complete Software Stack

Beyond the FPGA fabric, the project features a production-ready C++ software stack running on the HPS.

A custom CLI tool communicating with the daemon via UNIX sockets for real-time management and status monitoring.

Autonomous Daemon

Systemd-managed C++ service with auto-recovery watchdogs. Handles camera failures and FPGA timeouts automatically.

Layered API

Clean abstraction layers separating low-level memory mapping (/dev/mem) from high-level application logic.

Key Engineering Challenges Solved

Avalon Bridge 4-Part Writes

The bridge split 128-bit transfers into four 32-bit chunks. Resolved by implementing internal counters in the RX/TX units to validate data only on every 4th cycle.

Legacy OS Limitations

Vendor Linux was outdated. Built a custom Arch Linux ARM distro with updated Kernel and Device Tree to support modern OpenCV and NCNN libraries.
